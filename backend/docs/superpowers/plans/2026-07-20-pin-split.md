# Pin Table Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split the overloaded `Pin` table into a venue-only `Pin` plus a new `ItineraryStop` table, add per-day `hoursOpen`, and replace the derived `tags` array with explicit `interests`/`cuisines`/`diets`/`category` fields — without changing the API response shape.

**Architecture:** Additive-first migration. Add the new table + columns alongside the existing ones, backfill data, switch reads, switch writes, then drop the old columns last. The itinerary read path reshapes `ItineraryStop + Pin` back into the flat `pins[]` array the frontend already consumes, so no frontend change is required. Full design: `docs/superpowers/specs/2026-07-20-pin-split-design.md`.

**Tech Stack:** Node ESM, Express, Prisma + PostgreSQL (Supabase), `node:test` (`npm test` = `node --test`), Vitest-free.

## Global Constraints

- Backend is ESM (`"type": "module"`); use `import`/`export`, `.js` extensions in import paths — copied from `backend/CLAUDE.md`.
- All DB access goes through a model (`models/*.js`); controllers/services never call `prisma` directly — from `.claude/rules/backend.md`.
- Tests are co-located `*.test.js`; run with `npm test` from `backend/`. Zero new test deps.
- Never commit directly to `main`; work on a branch. No `Co-Authored-By` trailer — from `.claude/rules/git.md`.
- Destructive migrations (dropping columns/data) can't run via non-interactive `prisma migrate dev`; hand-write the migration SQL and apply with `npx prisma migrate deploy` (as was done for `remove_like_count`).
- Migrations run from `backend/` (that's where `prisma/schema.prisma` + `.env` live).
- The full suite is currently **208 tests green** — every task must keep it green.

## Resolved design decisions (from spec open questions)

1. `mealType` — reshape appends it to the reconstructed `tags` array (frontend badge unchanged) AND exposes a `mealType` field.
2. `tripDate` — optional on `/recommendations`; defaults to a fixed day (`'2026-01-01'`, matching `persist.js`) when omitted.
3. `hoursOpen` backfill — every catalog pin starts with neutral `{ mon..sun: "08:00-22:00" }`; real hours refined later.
4. `category` — explicit `Pin.category` column, not derived.

```js
// backend/utils/hours.js
// Parses the per-day hoursOpen JSON ({ mon: "08:00-22:00", ..., sun: null })
// into the interval shape the recommendation engine already consumes.
// Three outcomes, matching the engine's missing-data rules:
//   - a day with a valid "HH:MM-HH:MM" range → [{ open, close }]
//   - a day explicitly null                  → null  (definitely closed)
//   - hoursOpen absent, day absent, or bad   → undefined (unknown ⇒ keep upstream)
const DAY_KEYS = ['sun', 'mon', 'tue', 'wed', 'thu', 'fri', 'sat']
const RANGE_RE = /^([01]\d|2[0-3]):[0-5]\d-([01]\d|2[0-3]):[0-5]\d$/

function dayKeyFromDate(dateISO) {
  // Parse as a Pacific-local calendar date; noon avoids any tz edge flipping the day.
  const probe = new Date(`${dateISO}T12:00:00-08:00`)
  return DAY_KEYS[probe.getUTCDay()]
}

function parseDayHours(hoursOpen, dayKey) {
  if (!hoursOpen || typeof hoursOpen !== 'object') return undefined
  if (!(dayKey in hoursOpen)) return undefined
  const value = hoursOpen[dayKey]
  if (value === null) return null
  if (typeof value !== 'string' || !RANGE_RE.test(value)) return undefined
  const [open, close] = value.split('-')
  return [{ open, close }]
}

export { parseDayHours, dayKeyFromDate }
```

Note on `dayKeyFromDate`: `getUTCDay()` on a `-08:00` noon instant returns the intended weekday because noon Pacific never crosses a UTC day boundary that changes the weekday. If the Thursday assertion fails, adjust the probe hour, not the test.

```js
import prisma from '../lib/prisma.js'

function createMany(stops) {
  return prisma.itineraryStop.createMany({ data: stops })
}

function findByItinerary(itineraryId) {
  return prisma.itineraryStop.findMany({
    where: { itineraryId },
    orderBy: { orderInItinerary: 'asc' },
    include: { pin: true },

}

export { createMany, findByItinerary, deleteByItinerary }
```

```js
// backend/scripts/backfillHoursOpen.js
// One-off, idempotent: give every Pin a neutral all-week open window so the
// engine can read real hours once it switches off the startTime/endTime proxy.
// Real per-day hours can be refined later.
import prisma from '../lib/prisma.js'

const NEUTRAL = {
  mon: '08:00-22:00', tue: '08:00-22:00', wed: '08:00-22:00',
  thu: '08:00-22:00', fri: '08:00-22:00', sat: '08:00-22:00', sun: '08:00-22:00',
}

async function main() {
  const result = await prisma.pin.updateMany({
    where: { hoursOpen: { equals: null } },
    data: { hoursOpen: NEUTRAL },
  })
  console.log(`Set hoursOpen on ${result.count} pins.`)
}

main().then(() => process.exit(0))
```
